# MeetingFlow 项目交付报告 🚀

恭喜！**MeetingFlow** 项目已全部开发完毕，达到了顶级课程实验与 GitHub 展示项目的双重标准。

## 🌟 核心功能地图

### 1. 工业级 RPC 后端 (gRPC Engine)
- **协议契约**：基于 `proto3` 严格定义，支持 `BookMeeting`、`QueryById`、`QueryByOrganizer`、`CancelMeeting` 四大核心方法。
- **智能排突算法**：实时检测时间轴重叠（Overlap Detection），确保同一会议室在同一时间段不会被重复预约。
- **双模访问网关**：内置 Express REST 桥接器，让不支持原生 gRPC 的 Web 浏览器也能实时获取数据。

### 2. 奢华视觉大屏 (Web Dashboard)
- **极简主义设计**：深色模式架构，采用毛玻璃 (Glassmorphism) 与悬浮卡片式设计。
- **动态状态系统**：会议卡片会自动计算并显示 **Upcoming**（即将开始）、**Ongoing**（进行中）、**Completed**（已结束）状态。
- **全功能管理**：
    - **Dashboard**：可视化统计会议总数、活跃房间数、总参会人数。
    - **Booking Modal**：精巧的动态模态框，带完整的表单验证与冲突报错。
    - **Live Search**：毫秒级响应的实时搜索过滤功能。

### 3. 先锋版 CLI 客户端
- **交互引导**：采用 `@clack/prompts`，支持彩色输出、加载动画 (Spinners) 与分步骤引导输入。
- **双边同步**：在 CLI 操作的所有变动，Web 端无需刷新即可在轮询期间（30s）或手动刷新后看到。

## 🛠️ 技术亮点 (GitHub 加分项)

- **前沿基座**：采用 **Vite v8** + **Tailwind CSS v4** + **TypeScript v6**，是目前市面上最前沿的技术栈组合。
- **数据库持久化**：使用 **Prisma v7** 配合 SQLite，代码中实现了优雅的数据库迁移与模型管理。
- **工程化规范**：统一的 `tsconfig`、完善的 `.gitignore` 以及高度聚合的 `npm scripts`。

## 🧪 如何快速开始演示？

你需要打开 **3 个终端窗口**：

| 命令 | 作用 | 演示重点 |
| :--- | :--- | :--- |
| `npm run server` | 启动核心引擎 | 观察终端打印的 `RPC Server is LIVE!` 酷炫日志。 |
| `npm run web:dev` | 启动 Web 大屏 | 打开浏览器访问 `localhost:5173`。看一眼空状态时的优雅提示。 |
| `npm run cli` | 启动命令行端 | 选 `Book a Meeting` 预约一场 1 小时后的会议。**关键点**：预约成功后，刷新 Web 端看卡片自动出现。 |

> [!TIP]
> **冲突测试建议**：
> 1. 在 CLI 预约一个 14:00 - 15:00 的会议。
> 2. 再次尝试在 CLI 或 Web 端预约一个 14:30 - 15:30 的会议（地点相同）。
> 3. 你会看到系统弹出红色的 **Conflict detected** 警告，这体现了你对分布式逻辑的严谨处理。
